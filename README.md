# SP500-MDA-Sentiment-Analysis
NLP analysis of CEO narrative sentiment in S&P 500 annual 10-K filings and its relationship with subsequent stock returns / Problem Set #2

This project analyses whether the language used by CEOs in their annual reports can predict how a company's stock will perform the following year. We extract the Management Discussion and Analysis (MD&A) section from over 400 filings across 68 S&P 500 companies, score the sentiment using three different NLP methods (VADER, Loughran-McDonald, and FinBERT), discover hidden themes using topic modelling, and test whether any of this language signal predicts returns.

## How to Run the Analysis

### Step 1 - Open the Project Folder

Open your IDE (Antigravity, VS Code, Jupyter, Anaconda, etc.) and ensure you have the root project folder open (the one containing `requirements.txt`).

### Step 2 - Download the Loughran-McDonald Dictionary

This file is required for the LM sentiment analysis in Phase 3. It is publicly available from the University of Notre Dame website and is included here solely for reproducibility of the research pipeline. Below are the steps to download directly, if needed.

1. Go to: https://sraf.nd.edu/loughranmcdonald-master-dictionary/
2. Fill in the short access form (name and email)
3. Download: `LoughranMcDonald_MasterDictionary_1993-2024.csv`
4. Rename it to: `Loughran-McDonald_MasterDictionary_1993-2024.csv`
5. Place it inside the `data/` folder

The notebook will run without this file but LM sentiment scores will be skipped and only VADER and FinBERT results will be produced.

### Step 3 - Install Dependencies

Open your terminal and run the appropriate command based on your current path:

- **If your terminal is in the main project folder:**
  `pip install -r requirements.txt`

- **If your terminal is inside the notebooks folder:**
  `pip install -r ../requirements.txt`

- **If using a cloud notebook (Anaconda, Jupyter Hub, etc.):**
  Run `%pip install -r requirements.txt` in the first cell of the notebook.

### Step 4 - Open and Execute the Main Notebook

1. Navigate to `notebooks/` and open `01_SP500_Sentiment_Analysis.ipynb`
2. Select your Python kernel (Python 3.10 or above recommended)
3. Click **Kernel -> Restart and Run All**

The notebook loads all pre-built data from the `data/` folder. No internet connection or scraping is required for the main analysis.

## Runtime Guide

Most cells complete in seconds. The two cells below are the exception and should be left to run uninterrupted.

> **WARNING - Phase 4 (FinBERT Scoring): approximately 1-2 hours on CPU, 15-20 minutes on GPU.**
> This cell downloads the FinBERT model weights (~440MB) on the first run and scores all 417 documents sentence by sentence. Do not interrupt this cell. Results are saved to `data/sentiment_finbert.csv` automatically. All subsequent runs will load from this cache and complete in seconds.

> **WARNING - Phase 5 (LDA Coherence Validation): approximately 45-60 minutes.**
> This cell fits 16 separate LDA models across k = 5 to 20 to identify the optimal number of topics. The fixed seed (random_state = 42) guarantees identical results on every run. Do not interrupt this cell. If the kernel appears to hang for more than 5 minutes without any output, wait - this cell prints one line per k value so output is intermittent.

All other phases complete in under 2 minutes each. Total first-run time is approximately 2-3 hours. Once both cache files exist, the full notebook reruns in under 10 minutes.

## Troubleshooting and Reproducibility

**FinBERT warning about position_ids:**
When loading FinBERT you may see a message about `bert.embeddings.position_ids` being unexpected. This is a known compatibility note between newer versions of the transformers library and older model checkpoints. It does not affect results and can be safely ignored.

**HuggingFace unauthenticated request warning:**
You may see a warning: "You are sending unauthenticated requests to the HF Hub." This does not affect functionality. FinBERT will still download and run correctly without an API token.

**LM dictionary not found:**
If Phase 3 prints "LM dictionary not loaded", the file is either missing from `data/` or the filename does not match exactly. Check that the file is named `Loughran-McDonald_MasterDictionary_1993-2024.csv` with the hyphen between Loughran and McDonald. Delete `data/sentiment_vader_lm.csv` and re-run Phase 3 after fixing the filename.

**yfinance MultiIndex error:**
If the stock returns cell throws a KeyError on "last" or "Close", your yfinance version uses a MultiIndex column format. The notebook handles this automatically. If you encounter it, delete `data/stock_returns.csv` and re-run the returns cell.

**Kernel hangs or cells spin indefinitely:**
If a cell shows a running indicator for more than 10 minutes without output (excluding the FinBERT and LDA cells which are expected to be slow), interrupt the kernel, select "Clear All Outputs", and restart. Ensure you are running Python 3.10 or above.

**Fixed Seed:**
All models use `random_state = 42` to ensure reported metrics, topic counts, and correlation values are identical across all hardware and platforms.

**Relative Paths:**
All file paths in the notebook are relative (e.g., `../data/`). Do not move the notebook out of the `notebooks/` folder or paths will break.

The data files provided in this repository were built using `00_Data_Generation.ipynb`. You do not need to run this notebook to reproduce the analysis. It is included for full methodological transparency so that anyone can independently verify how the MD&A corpus was extracted from SEC EDGAR.

> **WARNING - Data generation runtime: approximately 35-70 minutes total.**
> Phase 1 (filing index) takes 5-10 minutes. Phase 2 (MD&A extraction) takes 30-60 minutes as it downloads 400+ individual filings from SEC EDGAR one at a time. Both phases cache their outputs and skip on re-run.

If you choose to regenerate the corpus, the extracted `mda_corpus.csv` should contain 417 documents. Minor variation (plus or minus a few documents) is possible if EDGAR filing availability has changed since the original extraction.

## Key Results

The analysis finds that MD&A sentiment does not significantly predict subsequent annual stock returns across any of the three sentiment methods tested. Lagged Pearson correlations are near zero for VADER (r = -0.024), LM Net Sentiment (r = +0.033), and FinBERT (r = +0.026), all well below the minimum detectable effect of r = 0.150 at 80% statistical power. This is consistent with the efficient market hypothesis: information in publicly disclosed narratives is rapidly incorporated into prices before the annual measurement window.

The LDA topic model identified 16 latent themes (optimal k selected via C_v coherence validation), including sector-specific topics for Energy, Healthcare, and Financial Services, with risk-related topics showing elevated prevalence during COVID-2020.
